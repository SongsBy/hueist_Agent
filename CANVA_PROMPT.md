## Slide 1 — 전체 구현 & 흐름
- **제목**: Next.js로 만든 AI 디자인 톤 생성 웹
- 프론트 + 백엔드 API를 한 프로젝트에 통합 (별도 서버 없음)
- 4단계 흐름 (화살표 다이어그램으로):
  `① 템플릿 선택 → ② 원하는 앱 입력 → ③ 톤 추천 → ④ UI 생성`
- 선택한 톤·입력·템플릿은 전역 상태로 저장되어 페이지 간 유지

## Slide 2 — 템플릿: 미리 만들어 둔 JSX
- **제목**: 검증된 레이아웃을 코드로 미리 준비
- 도메인별 모바일 화면 레이아웃 **14종**을 JSX로 사전 제작 (피드·금융대시보드·커머스·배달·메신저·뮤직 등)
- 사용자가 1장 선택 → 템플릿 id 저장
- 이 JSX 코드는 **마지막 UI 생성 단계에서 AI에게 '설계도'로 주입** (단순 미리보기가 아님)
- → AI가 레이아웃을 새로 상상하지 않고 검증된 구조 위에서 작업

## Slide 3 — 톤 추천은 어떻게 만들었나
- **제목**: 확실히 다른 톤 3장 — COLOR STRATEGY MATRIX
- `/api/recommend`가 Google **Gemini**로 톤 3개 JSON 생성
- 3개 톤에 배색 전략 **1:1 강제**: ① 모노크로매틱 ② 미니멀 틴트 ③ 보색 파격
- "같은 색을 3톤에 반복하면 응답 무효" 명시 → 선택지가 진짜 갈리게 보장
- 설문(카테고리·무드·색 방향)을 서술형 한국어 브리프로 변환, 색은 chroma-js로 명암 스케일 계산

## Slide 4 — UI 생성: 모든 게 합쳐지는 곳
- **제목**: 톤 + 입력 + 템플릿 + 외부 API → 실제 화면
- 합류 다이어그램으로 표현:
  - **외부 API 보강**(`/api/enrich`): Google Fonts(폰트) · Iconify(아이콘) · Pexels(이미지) 병렬
  - **`/api/generate-ui`**: ①보강된 톤 + ②사용자 앱 입력 + ③선택한 템플릿 JSX + ④지시 프롬프트를 합쳐 생성
- 프롬프트 = **시스템 프롬프트**(런타임 규칙) + **유저 프롬프트**(작업 지시)
  - 템플릿 선택 시 → "레이아웃 유지, 톤으로 재색칠만" 템플릿 모드 / 미선택 시 → 자유 생성 모드
- 모델: **Gemini 2.5 Flash**(1순위, 빠름) → 3 Flash 폴백, 작업별 temperature 차등
- 결과 JSX를 **react-live로 즉시 렌더**

## Slide 5 — 전처리기 + 린터 (신뢰성 장치)
- **제목**: AI 출력을 그냥 믿지 않는다
- 파이프라인 다이어그램: `템플릿 JSX → [전처리기] → 청사진 → Gemini → [린터] ↺ → 화면`
- **전처리기**: 템플릿의 Tailwind 클래스를 **인라인 스타일로 결정론적 변환** (런타임이 Tailwind 미컴파일 → AI는 번역 대신 톤 입히기에만 집중)
- **린터**: 생성 코드 검증 (인라인 스타일만·톤 색만·미정의 아이콘 금지·render() 필수)
- 위반 시 **사유를 피드백해 최대 3회 교정 재생성** + 색만 틀리면 근사 토큰 자동 보정
- → "AI를 그냥 호출하지 않고, **설계로 신뢰성을 만든** 도구"
